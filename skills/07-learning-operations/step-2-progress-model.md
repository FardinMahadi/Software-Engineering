# Step 2: Progress model

### Activity: Make progress measurable

Create `learning/progress-model.md` and define how progress is tracked across the vault.

Include:

1. A status model such as `not-started`, `in-progress`, `blocked`, `done`
2. Required frontmatter fields for operational notes
3. Rules for when to update `progress/current-focus.md`
4. Rules for when to update `progress/skill-tracker.md`
5. A definition of what counts as evidence of progress

Keep the model lightweight enough that you will actually maintain it.
