# Review Rhythm

Parent: [[progress/README|Learning Progress Dashboard]]

## Daily

1. Read [[progress/current-focus|Current Focus]] before starting work.
2. Complete one smallest useful action tied to the active skill.
3. Log blockers immediately in [[progress/weekly-review|Weekly Review]] or the active note.
4. If a step is completed, update [[progress/skill-tracker|Skill Tracker]] the same day.

## Weekly

Run a weekly review cycle on the review date in `progress/current-focus.md`:

1. Complete all prompts in [[progress/weekly-review|Weekly Review]].
2. Sync status, percentages, and next actions in [[progress/skill-tracker|Skill Tracker]].
3. Move non-urgent items to [[progress/learning-backlog|Learning Backlog]].
4. Confirm next week's top 1 to 3 actions in [[progress/current-focus|Current Focus]].

## Weekly review questions

1. What moved forward this week?
2. What slowed progress down?
3. Which files, deliverables, or commits prove progress happened?
4. Which topic is still weak enough that AI suggestions could mislead me?
5. What are the top 3 actions for next week?

## Backlog to focus movement rule

Move an item from backlog to focus only if all are true:

1. It supports the current active stage or resolves a current blocker.
2. It has a clear first action that can be done in one session.
3. It does not force more than one skill to be `in-progress` at once.

If one of these is false, keep it in backlog.

## Depth decision rule (good enough vs deeper study)

A topic is "good enough for now" if:

1. You can explain it in your own words.
2. You applied it to a file, task, or decision.
3. You can define one failure mode or tradeoff.

If any point is missing, schedule deeper study in backlog.

## Rule

If a note does not change the next action, it should probably not be edited during a quick review.
