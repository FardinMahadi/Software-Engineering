# Step 3: Review rhythm

### Activity: Build the review loop

Create `learning/review-rhythm.md` and design a repeatable review cadence.

Your note should define:

1. What gets updated daily
2. What gets updated weekly
3. Which questions you ask during review
4. How incomplete tasks move from backlog to focus
5. How to decide whether a topic needs more depth or is good enough for now

Use `progress/weekly-review.md` as the operational companion note and keep all examples Obsidian-friendly.
