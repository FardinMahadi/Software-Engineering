# 📘 Software Engineering — Self-Study Guide

> A personal learning repository for Software Engineering concepts, following the structured course guide at [Software Engineering Full Course](https://www.youtube.com/watch?v=oshQg1uSRvg).

---

## 🎯 Purpose

This repository serves as a structured study companion for learning **Software Engineering** from the ground up. It tracks notes, diagrams, exercises, and project artifacts organized by topic, making it easy to revisit concepts and apply them in practice.

---

## 📚 Topics Covered

### 1. Introduction to Software Engineering
- What is Software Engineering?
- Software vs. Program
- Characteristics of good software
- Software Engineering principles

### 2. Software Development Life Cycle (SDLC)
- Planning
- Requirements Analysis
- System Design
- Implementation / Coding
- Testing
- Deployment
- Maintenance

### 3. SDLC Models
- Waterfall Model
- Iterative Model
- Spiral Model
- V-Model (Verification & Validation)
- Agile Model
- RAD (Rapid Application Development)

### 4. Requirements Engineering
- Functional vs. Non-Functional Requirements
- Requirements Elicitation Techniques
- Use Case Diagrams
- Software Requirements Specification (SRS)

### 5. Software Design
- Levels of Design (Architectural, High-Level, Low-Level)
- Modular Design
- Coupling & Cohesion
- Design Patterns (Creational, Structural, Behavioral)
- UML Diagrams:
  - Class Diagram
  - Sequence Diagram
  - Activity Diagram
  - State Machine Diagram

### 6. Software Testing
- Levels of Testing: Unit → Integration → System → Acceptance
- Types: Black-box, White-box, Grey-box
- Test Cases & Test Plans
- Regression Testing
- Test-Driven Development (TDD)

### 7. Software Project Management
- Project Scheduling (Gantt Charts, PERT/CPM)
- Effort & Cost Estimation (COCOMO Model)
- Risk Management
- Configuration Management

### 8. Agile & Scrum
- Agile Manifesto & 12 Principles
- Scrum Framework (Sprints, Backlog, Standups)
- Kanban
- Extreme Programming (XP)

### 9. Software Quality Assurance (SQA)
- Quality Metrics (Reliability, Maintainability, Usability)
- Code Reviews & Inspections
- ISO 9001 & CMMI

### 10. Version Control
- Git Basics (clone, add, commit, push, pull)
- Branching Strategies
- Pull Requests & Code Review Workflow
- GitHub / GitLab Workflows

- `diagrams/` — UML and other visual aids
- `exercises/` — practice problems and solutions

---

## 🛠️ How to Use This Repo

1. **Watch** the corresponding video section from the [course guide](https://www.youtube.com/watch?v=oshQg1uSRvg).
2. **Read** the notes in the relevant topic folder.
3. **Practice** by completing the exercises in that folder.
4. **Commit** your own notes and solutions to track your progress.
